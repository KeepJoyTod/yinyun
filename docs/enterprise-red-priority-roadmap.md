## 企业版实现策略

正式给企业使用时，建议以 RuoYi-Vue-Plus 为企业后台基座，但第一阶段只实现上述 7 条，不先铺开商城、会员、营销、储值、支付等大模块。

### 阶段 1：平台底座

- 初始化 RuoYi-Vue-Plus。
- 保留系统能力：用户、角色、菜单、字典、租户、操作日志、登录日志。
- 建立业务模块前缀：`yy_store`、`yy_product`、`yy_order`、`yy_asset`、`yy_channel`、`yy_report`。
- 菜单先只开：首页、门店管理、产品管理、预约订单、客片选片、渠道插件。

### 阶段 2：核心数据模型

- `yy_store`：门店名称、营业状态、地址、电话、本月订单数、待服务单数。
- `yy_product`：产品名称、产品类型、服务组、价格、时长、选片单价、入册产品。
- `yy_channel_plugin`：渠道类型、租户/品牌、开通状态、提示文案。
- `yy_channel_account`：渠道授权账号、`app_key`、`access_token`、过期时间、授权状态。
- `yy_channel_order_mapping`：外部订单号、本地订单号、外部状态、同步状态、原始 payload。
- `yy_channel_sync_log`：接口名、请求结果、错误信息、耗时、重试标记。
- `yy_order`：订单号、门店、客户、手机号、来源、预约方式、下单时间、到店时间、状态。
- `yy_photo_album` / `yy_photo_asset`：相册、底片、排序、是否已选、公开选片状态。

### 阶段 3：第一批页面

- 预约订单列表：先做筛选、列表、状态标签、导出预留。
- 首页预约概况：订单状态、时段工位、预约趋势。
- 门店管理：门店卡片、本月订单、待服务单、营业状态、地址资料。
- 产品管理：选片单价、入册产品、抖音/美团产品类型。
- 客片选片：底片列表、是否已选、客户可见入口。
- 渠道插件：未授权时显示抖音/美团未开通；抖音优先预留服务市场已购查询、购买明细和 `service_market_order` webhook；美团等待核销接口文档后补 adapter。

### 阶段 4：渠道插件对接

详细方案见 `docs/channel-plugin-integration-plan.md`。

- 抖音第一版先做只读查询：服务市场购买状态、购买明细、同步日志。
- webhook 事件先进入渠道订单池，人工确认后再生成本地预约订单。
- token 与原始 payload 做权限隔离，普通员工不可查看敏感字段。
- 美团使用同一套插件模型，先实现未开通和 mock adapter，不阻塞抖音试点。

## 竞争交付建议

先用当前 Next MVP 做功能对照演示，再用 RuoYi-Vue-Plus 企业版做正式重构试点。对外演示时不要强调“技术栈”，重点展示：

- 旧站标红功能已经逐项还原。
- 企业版有用户、角色、菜单、日志、租户和数据权限底座。
- 抖音/美团不是简单写死按钮，而是插件授权能力。
- 订单列表和预约概况优先可用，能支撑真实门店试用。
