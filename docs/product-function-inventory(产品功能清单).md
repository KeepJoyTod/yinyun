# 预约 SaaS 产品功能清单与实现状态

生成时间：2026-05-29  
适用范围：消费者端、商户管理后台、会员/卡券/储值、服务协作、报表与平台运维能力。  
状态口径：本文区分“参考系统观察态”和“当前项目落地态”。“参考系统已观察/当前项目待核查”只表示参考系统在体验账号下可见菜单、页面、字段或动作入口，不代表当前仓库已完成；“当前项目已完成/派生只读/API 只读/脚手架/建设中”表示当前仓库已有对应 owner、接口、只读聚合或骨架，但仍需按说明判断是否达到生产闭环；“待实现”表示当前仓库未发现可验收 owner 或真实业务链路。本文不记录测试账号密码、客户手机号、证件号、地址等敏感信息。
最近补充验证：2026-06-24，重点补充商户后台侧栏、店铺装修、产品、订单、服务生产、会员、工具、统计、设置、应用商城、费用中心和账号中心页面的可见功能入口。
当前项目标注口径：状态包含“当前项目已完成”的条目，基于 `docs/yiyue/function_map.md`、`docs/studio-workbench-route-implementation-status.md`、`studio-workbench/src/app/router/featureRegistry.ts` 和后端 controller/service 中已落地的 owner、接口或脚手架；“实现时间”填写对应任务包落地日期。带有“只读、派生、脚手架、建设中、待验收”的条目不等同于生产闭环完成。

## 1. 状态定义

| 状态 | 判定口径 | 产品处理建议 |
|---|---|---|
| 参考系统已观察/当前项目待核查 | 参考系统账号可见菜单、页面、列表、字段、按钮或可进入流程，但当前仓库未确认同等 owner/API | 只能作为需求基线，不能按当前项目完成统计 |
| 当前项目已完成 | 当前仓库已有正式 owner、接口或真实数据链路，并有对应任务包落地记录 | 进入验收清单，继续补权限、审计、异常、导出和生产 smoke |
| 当前项目已完成/派生只读/API 只读/脚手架/建设中 | 当前仓库已有入口、只读聚合、派生页、接口 facade 或骨架，但写链路/真实授权/生产闭环未完成 | 保留为部分落地，后续按 owner 补真实写链路和端到端验收 |
| 部分实现/待确认 | 入口已观察，但未执行保存、支付、退款、核销、发券等会改变数据的动作 | 在测试环境补端到端用例，确认真实状态机和数据落库 |
| 未开通/未授权 | 当前账号点击后提示未开通，或路由存在但页面不可访问 | 作为插件/套餐能力管理，设计授权开关和灰度策略 |
| 待实现 | 商业级 SaaS 必需，但参考系统当前未观察到完整产品化能力 | 纳入新系统版本规划，明确最小可交付范围 |
| 合规/风控待确认 | 涉及储值、提现、退款、发票、短信、隐私、支付清结算 | 上线前需要法务、财务、运营共同确认 |
