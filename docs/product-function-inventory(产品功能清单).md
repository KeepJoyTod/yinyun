# 预约 SaaS 产品功能清单与实现状态

生成时间：2026-05-29  
适用范围：消费者端、商户管理后台、会员/卡券/储值、服务协作、报表与平台运维能力。  
状态口径：本文的“已实现”指参考系统在当前体验账号下已观察到菜单、页面、字段或动作入口；“未实现/待实现”指商业化 SaaS 需要补齐但当前未观察到完整能力；“未开通/未授权”指菜单、路由或入口存在，但当前账号或租户套餐不可用。本文不记录测试账号密码、客户手机号、证件号、地址等敏感信息。
最近补充验证：2026-06-16，重点补充商户后台侧栏、店铺装修、产品、订单、服务生产、会员、工具和设置页面的可见功能入口。

## 1. 状态定义

| 状态 | 判定口径 | 产品处理建议 |
|---|---|---|
| 已实现/已观察 | 当前账号可见菜单、页面、列表、字段、按钮或可进入流程 | 进入需求基线，按商业版标准补齐权限、审计、异常、导出和验收 |
| 部分实现/待确认 | 入口已观察，但未执行保存、支付、退款、核销、发券等会改变数据的动作 | 在测试环境补端到端用例，确认真实状态机和数据落库 |
| 未开通/未授权 | 当前账号点击后提示未开通，或路由存在但页面不可访问 | 作为插件/套餐能力管理，设计授权开关和灰度策略 |
| 待实现 | 商业级 SaaS 必需，但参考系统当前未观察到完整产品化能力 | 纳入新系统版本规划，明确最小可交付范围 |
| 合规/风控待确认 | 涉及储值、提现、退款、发票、短信、隐私、支付清结算 | 上线前需要法务、财务、运营共同确认 |
